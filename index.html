Two-brain workflow

Diagnostics is read-only. Remediation is approval-gated and includes rollback + validation.

1

Diagnostics (no approval required)

Collect facts, isolate layer/domain, propose likely causes, provide safe checks and evidence requests.

  • Symptoms → Likely Causes → Diagnostics → Interpretation
  • Expected outputs (“good vs bad” patterns)
  • Produces a proposed remediation plan
2

Remediation (approval required)

Only after explicit user intent + approval. Outputs change steps with risk, rollback, and validation.

  • Change Summary → Preconditions → Steps
  • Rollback/backout plan
  • Validation checklist + post-change notes

FAQ

Common questions about using the agent in real operational workflows.

Can the agent run commands on my systems?
No. It provides guidance, commands to run, and interpretation patterns. You remain in control of execution.
Why is remediation approval-gated?
To match enterprise change-control practices. Diagnostics remains read-only; remediation is only provided after explicit intent and approval.
Can I paste screenshots of errors?
Yes. Paste screenshots or text. The agent will extract key details and proceed through the appropriate troubleshooting workflow.
What should I save in tickets?
Copy the Diagnostics section (symptoms, scope, tests run, interpretations) and the Remediation section (steps, rollback, validation) when used.
